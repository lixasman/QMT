## 1. 系统定位与范式定义

### 1.1 范式转移（V1 → V2）

> **V1 定位**：提取微观因子 → 供下游预测方向性 Alpha
> **V2 定位**：刻画流动性状态与信息环境 → 供下游作为风险过滤器与波动率状态描述

在 Daily Batch 模式下（T-0 日 15:30 计算 → T+1 日使用），因子从计算到消费间隔 ≥ 18 小时。任何依赖"小时级预测力"的方向性信号在 T+1 日已基本衰减殆尽。因此 V2 将微观因子的角色严格定义为三类：

| 角色 | 代表因子 | 信号含义 |
|------|---------|---------|
| **波动率状态标识** | VPIN, ΔVPIN, RV_bipower | T+1 日发生极端行情或流动性枯竭的概率 |
| **流动性约束地图** | Kyle's Lambda | 真实交易摩擦，指导下游对拥挤品种降权 |
| **信息环境描述** | OFI 偏度, OFI-Price Divergence, BVC 净流 | 机构知情交易者在 T-0 日的建仓残留痕迹 |

### 1.2 执行模式：仅 Daily Batch

```
每个交易日 15:30 后执行：

15:30  通过 XtQuant 拉取今日历史 L1 快照 → 存入 Parquet（已由筹码引擎完成）
15:31  加载今日 L1 快照 Parquet
15:32  执行微观因子引擎 daily_batch → 输出每只 ETF 的因子值
15:33  合并筹码引擎输出 + 微观因子输出 → 最终因子矩阵

不涉及实时订阅、不涉及盘中计算。
```

### 1.3 硬件环境

与筹码引擎相同：Python 3.11 / NumPy / SciPy / Pandas。

### 4.1 volume/amount 口径检测与增量化

**问题**：XtQuant 推送的 `volume/amount` 可能是当日累计值而非快照增量。若误用累计值，BVC/VPIN/VS 全部失真。

```python
def detect_and_fix_volume(snapshots: pd.DataFrame) -> tuple[pd.DataFrame, dict]:
    """
    检测 volume 是否为累计口径，若是则差分为增量。
    
    检测规则：
    - 若 volume 序列单调非递减且最后值 >> 首值（>10倍），判定为累计口径
    - 差分后 clip(lower=0) 去除可能的负值异常
    
    Returns:
        (处理后 DataFrame, 元数据 dict: {"volume_is_cumulative": bool})
    """
    df = snapshots.copy()
    vol = df["volume"].astype(float)
    
    is_monotonic = (vol.diff().fillna(0) >= 0).mean() > 0.95
    ratio = vol.iloc[-1] / max(vol.iloc[0], 1) if len(vol) > 1 else 1
    is_cumulative = is_monotonic and ratio > 10
    
    if is_cumulative:
        df["volume"] = vol.diff().fillna(vol.iloc[0]).clip(lower=0).astype(int)
        if "amount" in df.columns:
            amt = df["amount"].astype(float)
            df["amount"] = amt.diff().fillna(amt.iloc[0]).clip(lower=0)
    
    return df, {"volume_is_cumulative": is_cumulative}
```

### 4.2b 时间戳去重（V2.1 新增）

> ⚠️ **V2.1 修正（第三轮审计 P1-1）**：生产数据源在异常日可能重复推送同一时间戳的快照。若不去重，重复数据会被当作真实成交消费，导致 BVC/VPIN/VWAP 等全部失真。

```python
def dedup_by_timestamp(snapshots: pd.DataFrame) -> tuple[pd.DataFrame, dict]:
    """
    去重策略：同一 timestamp 仅保留最后一条（累计口径时为最新值）。
    
    Returns:
        (DataFrame, {"duplicate_ts_count": int, "duplicate_ts_ratio": float})
    """
    n_before = len(snapshots)
    df = snapshots.drop_duplicates(subset=["timestamp"], keep="last")
    n_dup = n_before - len(df)
    return df, {"duplicate_ts_count": n_dup,
                "duplicate_ts_ratio": n_dup / max(n_before, 1)}
```

> 1. 新增 `close > 0`：防止停牌段/数据源异常的 close=0 泄漏到 RV Bipower 产生 `log(1e-8)≈-18.4` 的极端 log return

```python
def build_validity_mask(snapshots: pd.DataFrame) -> pd.Series:
    """
    有效条件（全部满足）：
      bid1 > 0, ask1 > 0, bid1 < ask1,
      bid1_vol > 0, ask1_vol > 0,  # V2.1: 从 >=0 收紧为 >0

    valid = (
        (snapshots["bid1"] > 0) &
        (snapshots["ask1"] > 0) &
        (snapshots["bid1"] < snapshots["ask1"]) &
        (snapshots["bid1_vol"] > 0) &   # V2.1: 从 >=0 改为 >0
        (snapshots["ask1_vol"] > 0) &   # V2.1: 从 >=0 改为 >0
        (snapshots["close"] > 0)        # V2.1: 新增
    )
```

## 5. Module M1: Microprice 基建

**文件**: `microstructure/microprice.py`
**职责**: 计算 Microprice 序列，作为 BVC 和 Lambda 的底层价格锚。**所有需要"价格变化"信号的模块，优先使用 Microprice 而非 close**。

> ⚠️ **V2.1 修正（红队 P1-4）**：新增涨跌停保护。当 ask1 ≤ 0（涨停）或 bid1 ≤ 0（跌停）时，midprice 会退化为 bid1/2 或 ask1/2，产生价格腰斩。保护逻辑使用 close 替代。

### 算法

Microprice（盘口挂单量加权均衡价格）：

$$M_t = \frac{Bid_1(t) \times AskVol_1(t) + Ask_1(t) \times BidVol_1(t)}{BidVol_1(t) + AskVol_1(t)}$$

**经济学含义**：当买一挂单量远大于卖一挂单量时，Microprice 向 Ask 偏移（隐含"买方更激进"）。即使实际成交价不变（dp=0），Microprice 变化也能反映订单簿的潜在压力。

### 接口

```python
def compute_microprice(snapshots: pd.DataFrame) -> np.ndarray:
    """
    计算 Microprice 序列（含涨跌停保护）。

    Args:
        snapshots: 含 bid1, bid1_vol, ask1, ask1_vol, close 列（经 M0 预处理）

    Returns:
        np.ndarray: 与 snapshots 等长的 Microprice 序列
    """
    bid_p = snapshots["bid1"].values
    bid_v = snapshots["bid1_vol"].values.astype(np.float64)
    ask_p = snapshots["ask1"].values
    ask_v = snapshots["ask1_vol"].values.astype(np.float64)
    close = snapshots["close"].values
    
    total_vol = bid_v + ask_v
    # 防止除零：当 total_vol==0 时退化为 midprice
    safe_total = np.where(total_vol > 0, total_vol, 1.0)
    microprice = (bid_p * ask_v + ask_p * bid_v) / safe_total
    # total_vol==0 时使用 midprice
    midprice = (bid_p + ask_p) / 2.0
    microprice = np.where(total_vol > 0, microprice, midprice)
    
    # ── 涨跌停保护（V2.1 红队修正）──
    # 涨停: ask1 <= 0 或 ask1_vol == 0 → midprice = (bid1 + 0)/2 = bid1/2（价格腰斩）
    # 跌停: bid1 <= 0 或 bid1_vol == 0 → midprice = (0 + ask1)/2 = ask1/2（价格腰斩）
    # 保护策略：使用 close 替代异常 microprice
    limit_mask = (ask_p <= 0) | (bid_p <= 0)
    microprice[limit_mask] = close[limit_mask]
    
    return microprice
```

## 6. Module M2: BVC 买卖分类器（Microprice 增强）

**文件**: `microstructure/bvc.py`
**职责**: 将每个快照的成交量分类为买入量和卖出量。**V2 引入三级退化补偿机制**。

### 算法：三级退化补偿

> ⚠️ **V2.1 修正（红队 P0-2 + P1-5）**：
> 1. Level 2 增加最小幅度阈值 `|dm| > 2 × TICK_SIZE`（0.002 元），过滤做市商微调挂单量产生的亚 tick Microprice 噪声。不满足阈值则跳至 Level 3。
> 2. σ_Signal 改为**按 Level 分别计算**独立滚动标准差，避免三种量级信号混合导致 Level 2 的 z 坍缩至 ≈0。

```
Signal_t 的选取逻辑（按优先级递降）：

Level 1: dp = close[t] - close[t-1]          若 dp ≠ 0 → 使用 dp
Level 2: dm = microprice[t] - microprice[t-1]  若 dp=0 且 |dm| > 2×TICK_SIZE → 使用 dm
Level 3: dp_k = close[t] - close[t-k]         向前追溯至 dp_k ≠ 0，k_max=10（30秒）
Fallback: 若三级全为 0 → Φ(0)=0.5，强制 50/50 分配
```

$$buy\_pct_t = \Phi\left(\frac{Signal_t}{\sigma_{Signal,level}}\right)$$

其中 $\sigma_{Signal,level}$ 为**该 Level 对应的 Signal 子序列**的滚动标准差（窗口=100）。每个 Level 独立维护 σ，避免量级混合。

### 接口

### 实现要点

```python
import math
import numpy as np

def _compute_signal(close: np.ndarray, microprice: np.ndarray,
                    k_max: int = 10, level2_min_dm: float = 0.002):
    """三级退化补偿信号计算（V2.1: Level 2 阈值过滤）"""
    n = len(close)
    signal = np.zeros(n)
    levels = np.zeros(n, dtype=int)  # 0=N/A, 1=L1, 2=L2, 3=L3, 4=fallback
    
    for t in range(1, n):
        # Level 1: close diff
        dp = close[t] - close[t - 1]
        if abs(dp) > 1e-9:
            signal[t] = dp
            levels[t] = 1
            continue
        
        # Level 2: microprice diff（V2.1: 需超过 2×tick 才使用）
        dm = microprice[t] - microprice[t - 1]
        if abs(dm) > level2_min_dm:  # 过滤做市商微调噪声
            signal[t] = dm
            levels[t] = 2
            continue
        
        # Level 3: 历史回溯
        found = False
        for k in range(2, min(k_max + 1, t + 1)):
            dp_k = close[t] - close[t - k]
            if abs(dp_k) > 1e-9:
                signal[t] = dp_k
                levels[t] = 3
                found = True
                break
        
        if not found:
            signal[t] = 0.0  # Φ(0) = 0.5
            levels[t] = 4
    
    return signal, levels

def _compute_per_level_sigma(signal: np.ndarray, levels: np.ndarray,
                             lookback: int = 100) -> np.ndarray:
    """按 Level 分别计算滚动标准差（V2.1 修正）
    
    各 Level 信号量级差异 10-100 倍（Level 1 ~0.001, Level 2 ~0.0001-0.0005, Level 3 ~0.001-0.01）。
    混合计算 σ 会导致 Level 2 的 z = dm/σ ≈ 0 → Φ(z) ≈ 0.5（实质失效）。
    
    解决方案：每个 Level 独立维护滚动 σ。对于某 Level 历史样本不足 5 个时，
    使用所有 Level 的 σ 作为 fallback。
    """
    n = len(signal)
    sigma = np.ones(n) * 1e-8  # 默认极小值
    
    for lv in [1, 2, 3]:
        lv_mask = levels == lv
        lv_indices = np.where(lv_mask)[0]
        if len(lv_indices) < 5:
            continue
        # 对该 Level 的信号子序列计算 expanding/rolling std
        lv_signals = signal[lv_mask]
        lv_std = pd.Series(lv_signals).rolling(lookback, min_periods=5).std().fillna(
            pd.Series(lv_signals).expanding(min_periods=5).std()
        ).values
        # 写回对应位置
        sigma[lv_indices] = np.maximum(lv_std, 1e-8)
    
    return sigma

def _phi(z: float) -> float:
    """标准正态 CDF，使用 math.erf"""
    return 0.5 * (1.0 + math.erf(z / math.sqrt(2.0)))
```

### bvc_quality 诊断指标

$$bvc\_quality_d = 1 - \frac{N_{fallback}}{N_{total}}$$

当 `bvc_quality < 0.3` 时，下游应自动降权该 ETF 当日的 VPIN 和 Lambda 因子。

### 算法

1. **桶大小**：$V_{bucket} = ADV_{60} / 50$
2. **桶构建**：将日内快照的 v_buy/v_sell 按顺序灌入等量桶，每桶装满 $V_{bucket}$ 后封桶
3. **桶级 OI**：$OI_\tau = |V_{B,\tau} - V_{S,\tau}| / V_{bucket}$
4. **VPIN**：对最近 50 个桶求滑动平均：$VPIN = \frac{1}{50} \sum_{\tau-49}^{\tau} OI_\tau$
5. **ΔVPIN**：$\Delta VPIN = VPIN_t - VPIN_{t-1}$（跨日差分，需历史存储）

> ⚠️ **V2.1 修正（二次评审 P1-8）**：原规格书仅声明参数名但未给出过滤的精确定义，导致实现不可复现。现明确如下：

```python
def _ap_filter_weight(premium_rate: float, threshold: float = 0.003,
                      saturate: float = 0.01) -> float:
    """
    AP 套利过滤权重：溢折价越高，该快照的成交量对 VPIN 的贡献越低。
    
    w = 1 - clip((|premium| - threshold) / (saturate - threshold), 0, 1)
    
    |premium| < threshold → w = 1.0（完全保留）
    |premium| > saturate  → w = 0.0（完全过滤）
    中间区间线性衰减
    
    应用方式：v_buy *= w, v_sell *= w（在灌桶前衰减成交量）
    仅影响 vpin_filtered，vpin_raw 必须保持不变。
    """
    excess = abs(premium_rate) - threshold
    if excess <= 0:
        return 1.0
    return float(1.0 - np.clip(excess / (saturate - threshold), 0, 1))
```

### 实现要点

```python
def _build_volume_buckets(v_buy, v_sell, volume, bucket_size):
    """
    将 v_buy/v_sell 序列灌入等量桶。
    每桶装满 bucket_size 后封桶，记录桶内的 V_B 和 V_S。
    最后不完整桶丢弃（论文标准做法）。
    
    V2.1 勘误：采用逐快照 buy_ratio 归因（非跨快照累积比例），
    保留 3 秒级 BVC 分类的时间分辨率，更接近 ELO 论文原意。
    """
    buckets_b, buckets_s = [], []
    current_buy, current_sell, accumulated = 0.0, 0.0, 0.0
    
    for i in range(len(volume)):
        remaining = volume[i]
        if remaining <= 0:
            continue
        snap_total = v_buy[i] + v_sell[i]
        buy_ratio = v_buy[i] / max(snap_total, 1e-12)
        
        while remaining > 0:
            space = bucket_size - accumulated
            fill = min(remaining, space)
            current_buy += fill * buy_ratio       # ← 保留当前快照的归因比例
            current_sell += fill * (1 - buy_ratio)
            accumulated += fill
            remaining -= fill
            
            if accumulated >= bucket_size - 1e-6:
                buckets_b.append(current_buy)
                buckets_s.append(current_sell)
                current_buy, current_sell, accumulated = 0.0, 0.0, 0.0
    
    return np.array(buckets_b), np.array(buckets_s)
```

```python
def tick_integerize(price: float, tick_size: float = 0.001) -> int:
    """
    将浮点价格转为 tick 整数，避免浮点 `==` 比较问题。
    
    舍入方式：四舍五入（round），与交易所价格处理一致。
    输出为整数（非浮点），消除累积漂移风险。
    
    示例：
        tick_integerize(1.001) = 1001
        tick_integerize(1.0015) = 1002  # 四舍五入
        tick_integerize(1.001, 0.01) = 100
    """
    return int(round(price / tick_size))

def ofi_price_changed(price_prev: float, price_curr: float,
                      tick_size: float = 0.001) -> bool:
    """判断价格是否变动（基于 tick 整数比较）"""
    return tick_integerize(price_prev, tick_size) != tick_integerize(price_curr, tick_size)
```

### OFI-Price Divergence（量价背离系数）

> ⚠️ **V2.1 修正（红队 P2-4）**：原公式使用 `Pearson(cumsum(OFI_t), Microprice_t)`，但 cumsum(OFI) 和 Microprice 均为非平稳序列，存在**伪回归风险**（spurious correlation），倾向于产生高绝对值 |ρ|。修正为使用差分后的同阶平稳序列。同时增加最小样本量控制。

$$\rho_{OFI,Price} = \text{Pearson}\left(OFI_t, \; \Delta Microprice_t\right)$$

其中 $\Delta Microprice_t = Microprice_t - Microprice_{t-1}$。差分后两序列均为同阶平稳，Pearson 相关系数具有统计意义。

**最小样本量控制**：当有效 OFI 快照（非 NaN）数量 < `OFI_DIVERGENCE_MIN_SAMPLES`（500）时，输出 0.0（中性值），避免小样本下 Pearson 的高方差导致虚假信号。

> ⚠️ **V2.1 修正（第三轮审计 P1-2）**：新增第二层门控——“有效价格变动样本数”。ETF 在 3s 粒度下大量 ΔMicroprice=0，即使总样本达 500，若非零价格变动点太少，Pearson 仍会退化为由 2-3 个跳点主导的极端值。增加：
>
> `count(|ΔMicroprice| >= TICK_SIZE) >= OFI_DIVERGENCE_MIN_MOVES`（50）
>
> 不满足时输出 0.0 并在 diagnostics 标记 `price_move_insufficient=1`。

经济学含义：若价格变动方向与 OFI 方向持续背离（负相关），说明价格上涨由缺乏流动性的跳空拉升构成，T+1 日存在均值回归动能。

## 9. Module M5: Kyle's Lambda 流动性冲击（诊断增强）

**文件**: `microstructure/kyle_lambda.py`
**职责**: 衡量市场深度与价格冲击成本。V2 使用 Microprice 替代 close diff，增加 R²/n_obs 诊断。

### 算法

> ⚠️ **V2.1 修正（二次评审 P1-6）**：原版使用无截距回归，但 ETF 价格变化包含 IOPV 漂移、基差回归等与 signed volume 无关的成分。无截距会把系统性漂移投影到 λ 上，导致偏置。修正为**带截距回归**。

$$\Delta M_t = \alpha + \lambda \times Q_t + \epsilon_t$$

其中 $\Delta M_t$ 为 Microprice 变化，$Q_t = V_{buy,t} - V_{sell,t}$（来自 BVC），$\alpha$ 为截距项（捕捉 IOPV 漂移/基差回归）。

使用日内尾部窗口 W=1200 快照（约 1 小时）执行**带截距**线性回归。

### 接口

```python
@dataclass
class LambdaResult:
    lambda_val: float        # λ 值（正值=正常，买压推升价格）
    intercept: float         # 截距 α（V2.1 新增，反映 IOPV 漂移）
    r_squared: float         # 回归 R²（可信度指标）
    n_obs: int               # 有效观测数量
    signed_vol_std: float    # SignedVol 标准差（退化检测）

def kyle_lambda(microprice: np.ndarray, v_buy: np.ndarray,
                v_sell: np.ndarray, window: int = 1200) -> LambdaResult:
    """
    计算日终的 Kyle's Lambda 及诊断指标。
    V2.1：使用带截距回归，避免 IOPV 漂移被误归因于 signed volume。
    """
    tail = min(window, len(microprice))
    dm = np.diff(microprice[-tail:])
    q = (v_buy[-tail:] - v_sell[-tail:])[1:]  # 对齐
    
    n_obs = len(dm)
    sv_std = float(np.std(q)) if len(q) > 0 else 0.0
    
    if n_obs < 50 or sv_std < 1e-8:
        return LambdaResult(float("nan"), float("nan"), float("nan"), n_obs, sv_std)
    
    # V2.1：带截距回归 dm = alpha + lambda * q
    # 第三轮审计 P0-1 修正：统一使用 population 统计量（ddof=0）
    # 原代码 np.cov(ddof=1) ÷ np.var(ddof=0) 导致确定性 N/(N-1) 偏差
    dm_mean = float(np.mean(dm))
    q_mean = float(np.mean(q))
    dm_centered = dm - dm_mean
    q_centered = q - q_mean
    
    # 手动计算 population covariance 和 variance，避免 ddof 不一致
    pop_cov = float(np.mean(dm_centered * q_centered))
    pop_var = float(np.mean(q_centered ** 2))
    lam = pop_cov / pop_var if pop_var > 1e-12 else float("nan")
    alpha = dm_mean - lam * q_mean if not np.isnan(lam) else float("nan")
    
    # R² 计算（带截距）
    predicted = alpha + lam * q
    ss_res = np.sum((dm - predicted) ** 2)
    ss_tot = np.sum((dm - dm_mean) ** 2)
    r2 = float(1.0 - ss_res / ss_tot) if ss_tot > 1e-12 else float("nan")
    
    return LambdaResult(lam, alpha, r2, n_obs, sv_std)
```

## 10. Module M6: 辅助因子（RV Bipower / VWAP 偏离度）

**文件**: `microstructure/auxiliary_factors.py`

### 10.1 已实现波动率 —— 跳跃惩罚 RV (Bipower Variation)

> ⚠️ **V2.1 修正（红队 P0-1）**：原版使用 Microprice 计算 log return，但 Microprice 受做市商挂单量波动驱动——做市商单边撤单时 Microprice 可瞬间偏移 1-2 个 tick（而 close 不变），频繁制造**伪跳跃**，导致 jump_ratio 被系统性高估 100-300%。修正为使用 **close 序列**计算 RV。

普通 RV 会被 Bid-Ask Bounce 严重污染。Bipower Variation 通过相邻收益率的乘积替代平方，自动过滤跳跃噪声。

$$BV_d = \frac{\pi}{2} \sum_{t=2}^{N} |r_t| \cdot |r_{t-1}|$$

其中 $r_t = \log(Close_t / Close_{t-1})$（**V2.1：使用 close 而非 Microprice**）。

```python
def rv_bipower(close: np.ndarray) -> dict:
    """
    使用 close 序列计算 RV，避免 Microprice 做市商挂单噪声制造伪跳跃。
    
    V2.1 第四轮修正：当有效 log_ret 数量不足时输出 NaN。
    零成交段 close 保持前值导致 log_ret=0，这是"无信息"而非"低波动"。
    
    Args:
        close: 快照 close 价格序列（经 M0 预处理）
    
    Returns:
        dict: {
            "rv_bipower": float,     # 跳跃惩罚已实现波动率（不足样本时 NaN）
            "rv_classical": float,   # 经典 RV
            "jump_ratio": float,     # 1 - BV/RV
            "rv_valid_count": int,   # V2.1 第四轮：有效 log_ret 数量
        }
    """
    MIN_RV_SAMPLES = 100  # 有效 log_ret 最少数量
    log_ret = np.diff(np.log(np.maximum(close, 1e-8)))
    valid_count = int(np.sum(np.abs(log_ret) > 0))
    
    if valid_count < MIN_RV_SAMPLES:
        return {"rv_bipower": float("nan"), "rv_classical": float("nan"),
                "jump_ratio": float("nan"), "rv_valid_count": valid_count}
    
    rv_classical = float(np.sum(log_ret ** 2))
    bv = float((np.pi / 2) * np.sum(np.abs(log_ret[1:]) * np.abs(log_ret[:-1])))
    jump = max(0.0, 1.0 - bv / rv_classical) if rv_classical > 1e-12 else 0.0
    return {"rv_bipower": bv, "rv_classical": rv_classical,
            "jump_ratio": jump, "rv_valid_count": valid_count}
```

### 10.2 VWAP 偏离度

$$VWAP\_dev = \frac{Close_{last} - VWAP}{VWAP}$$

其中 $VWAP = \sum(amount) / \sum(volume)$。

```python
def vwap_deviation(snapshots: pd.DataFrame) -> dict:
    """
    V2.1 修正（第三轮审计 P0-2）：原 clip(lower=1) 在 cum_volume=0 时
    产生天文级伪偏离。修正为：无成交区间 running_vwap/running_dev = NaN。
    
    Returns:
        dict: {
            "vwap_dev": float,         # 收盘价相对 VWAP 偏离度
            "vwap_dev_max": float,     # 日内最大偏离度（忽略 NaN）
            "zero_volume_ratio": float, # V2.1 新增：无成交快照占比
            "first_trade_index": int,   # V2.1 新增：首笔成交位置
        }
    """
    total_amount = snapshots["amount"].sum()
    total_volume = snapshots["volume"].sum()
    zero_volume_ratio = float((snapshots["volume"] == 0).mean())
    first_trade_idx = int((snapshots["volume"] > 0).idxmax()) if total_volume > 0 else -1
    
    if total_volume < 1:
        return {"vwap_dev": float("nan"), "vwap_dev_max": float("nan"),
                "zero_volume_ratio": zero_volume_ratio, "first_trade_index": first_trade_idx}
    
    vwap = total_amount / total_volume
    close_last = snapshots["close"].iloc[-1]
    vwap_dev = (close_last - vwap) / vwap if vwap > 1e-8 else 0.0
    
    # V2.1: 日内逐快照偏离度（无成交区间 = NaN，不再用 clip 伪造）
    cum_amount = snapshots["amount"].cumsum()
    cum_volume = snapshots["volume"].cumsum()
    has_volume = cum_volume > 0
    running_vwap = pd.Series(float("nan"), index=snapshots.index)
    running_vwap[has_volume] = cum_amount[has_volume] / cum_volume[has_volume]
    running_dev = (snapshots["close"] - running_vwap) / running_vwap
    vwap_dev_max = float(running_dev.abs().max(skipna=True))  # NaN 自动忽略
    if np.isnan(vwap_dev_max):
        vwap_dev_max = 0.0
    
    return {"vwap_dev": float(vwap_dev), "vwap_dev_max": vwap_dev_max,
```

### 10.3 Volume Surprise（V2.1 第四轮新增）

> ⚠️ **V2.1 修正（第四轮审计 D2-P0-1）**：M7 正交化公式使用 VS 作为基向量，但整部规格书的 M0-M6 从未定义或计算 VS。现明确定义如下。

$$VS = \log\left(\frac{V_{today}}{ADV_{60}}\right)$$

其中 $V_{today}$ 为当日总成交量，$ADV_{60}$ 为 60 日平均成交量。

```python
def volume_surprise(total_volume: float, adv_60: float) -> float:
    """
    Volume Surprise：度量当日成交量相对于历史均值的偏离程度。
    用于 M7 正交化（消除 VPIN 与成交量的同源性）。
    
    Returns:
        float: log(today_volume / adv_60)；当 adv_60 或 total_volume 无效时返回 NaN
    """
    if adv_60 <= 0 or total_volume <= 0:
        return float("nan")
    return float(np.log(total_volume / adv_60))
```

**经济学含义**：VS > 0 表示当日成交量高于历史均值，可能反映机构建仓/事件驱动；VS < 0 表示成交清淡。VPIN 与 VS 有结构性正相关（成交量越大，桶数越多，VPIN 统计精度越高），因此必须在正交化中剔除这层偏纯量能影响。

## 12. Module M8: 特征标准化管线（Robust Z + 截面排名）

**文件**: `microstructure/feature_pipeline.py`
**职责**: 将原始因子转换为标准化特征。V2 新增 Robust Z-Score 和截面标准化。

### 标准化方法对照表

| 因子 | 标准化方法 | 理由 |
|------|-----------|------|
| VPIN (正交化后) | 60 日 Rolling Rank | 有界分布，右偏 |
| ΔVPIN | Robust Z-Score (MAD) | 需保留方向性与极端值 |
| OFI skew | Robust Z-Score (MAD) | 厚尾分布 |
| OFI_95th | Robust Z-Score (MAD) | 厚尾分布 |
| OFI-Price Divergence | 直接使用（[-1,1]） + min_samples=500 + min_moves=50 | V2.1：双层门控 |
| Kyle's Lambda | Log + Robust Z-Score | V2.1 第四轮：右偏厚尾，线性 MAD 会截断 P95+ 信号 |
| RV_bipower | Log + Robust Z-Score | 右偏，V2.1：使用 close 计算 |
| VWAP_dev | Robust Z-Score (MAD) | 对称分布 |
| VS (Volume Surprise) | Robust Z-Score (MAD) | V2.1 第四轮新增 |

### 接口

> ⚠️ **V2.1 修正（两轮审计共同发现）**：
> 1. 冷启动处理：history 不足时输出 NaN（而非静默默认值 0.5/0.0），标记 `cold_start=1`
> 2. NaN/ties 处理：截面 rank 先剔除 NaN，对 ties 使用平均秩（二次评审 P3-1）
> 3. `cross_sectional_rank` 不在 per-ETF 内部调用，提升到全市场聚合层（二次评审 P0-2）

```python
from scipy.stats import rankdata

class FeaturePipeline:
    def __init__(self, history_days: int = 60):
        self.history_days = history_days

    def normalize_rank(self, current: float, history: np.ndarray) -> float:
        """Rolling Rank → [0, 1]
        V2.1: history 不足时输出 NaN（而非 0.5），由下游显式处理冷启动。
        """
        if len(history) < 5 or np.isnan(current):
            return float("nan")  # V2.1: 不再静默返回 0.5
        return float(np.sum(history < current) / len(history))

    def normalize_robust_zscore(self, current: float, history: np.ndarray,
                                clip_range: float = 3.0) -> float:
        """Robust Z-Score (MAD-based) → clipped to [-clip_range, clip_range]
        V2.1: history 不足时输出 NaN（而非 0.0）。
        """
        if len(history) < 5 or np.isnan(current):
            return float("nan")  # V2.1: 不再静默返回 0.0
        med = np.nanmedian(history)
        mad = np.nanmedian(np.abs(history - med))
        if mad < 1e-8:
            # V2.1 修正（第三轮审计 P0-1）：原返回 0.0 会把“平稳→突变”压扁为中性
            # 修正为：当 mad≈0 但 current 偏离 median 时，返回 ±clip_range（极端偏离）
            if abs(current - med) < 1e-10:
                return 0.0  # current ≈ med，真正的中性
            return float(np.sign(current - med) * clip_range)
        z = (current - med) / (1.4826 * mad)
        return float(np.clip(z, -clip_range, clip_range))

    def normalize_log_robust_zscore(self, current: float, history: np.ndarray) -> float:
        """Log1p + Robust Z-Score（右偏分布）"""
        log_c = np.log1p(max(current, 0))
        log_h = np.log1p(np.maximum(history, 0))
        return self.normalize_robust_zscore(log_c, log_h)

    def cross_sectional_rank(self, all_etf_values: dict[str, float]) -> dict[str, float]:
        """
        截面排名标准化：将当日所有 ETF 的同一因子值映射到 [-0.5, 0.5]。
        
        V2.1 修正：
          - NaN 先剔除，不参与排名，输出中保持 NaN
          - ties 使用平均秩（scipy.stats.rankdata method='average'）
        
        ★ 此方法不应在 per-ETF process_daily 内调用，
          应在全市场聚合层（daily_batch.py）调用。
        """
        codes = list(all_etf_values.keys())
        values = np.array([all_etf_values[c] for c in codes])
        
        # 分离 NaN 与有效值
        valid_mask = ~np.isnan(values)
        result = {c: float("nan") for c in codes}  # 默认 NaN
        
        valid_codes = [c for c, v in zip(codes, valid_mask) if v]
        valid_vals = values[valid_mask]
        
        if len(valid_vals) < 2:
            return result
        
        # 使用平均秩处理 ties
        ranks = rankdata(valid_vals, method="average")
        ranks = (ranks - 1) / max(len(ranks) - 1, 1) - 0.5  # 映射到 [-0.5, 0.5]
        
        for c, r in zip(valid_codes, ranks):
            result[c] = float(r)
        
        return result
```

## 16. 已知局限性声明

1. **订单轨迹坍缩**：3 秒切片内的 Spoofing（撤单→重挂→被吃）完全隐形，OFI 不可避免地低估真实博弈压力。升维路径：接入 Level 2/3 逐笔数据。

2. **流动性荒漠品种**：日均成交额 < 1000 万的微型 ETF，BVC 退化频率极高（bvc_quality < 0.3）。引擎通过 `bvc_quality` 指标如实暴露，下游应自动屏蔽该因子。

3. **T+1 预测周期天然错配**：18 小时时延注定无法捕捉毫秒至分钟级均值回归套利。下游必须将本引擎输出定位为"市场宏观状态过滤器"和"脆弱性预警器"，切忌作为高频买卖点触发器。

4. **Microprice 的做市商依赖与涨跌停保护**：Microprice 在做市商频繁调价时信号可靠；但在做市商撤离（流动性枯竭）时，盘口极度稀薄，Microprice 噪声急剧放大。此时 `bvc_quality` 会同步下降。V2.1 新增涨跌停保护（ask1=0 或 bid1=0 时使用 close 替代），防止 Microprice 跳变导致下游 BVC/Lambda 产生极端异常值。

5. **正交化的时序不稳定性**：VPIN 对 VS/RV 的回归系数可能随市场状态变化，正交化残差的含义会漂移。建议使用滚动窗口（60 日）而非全历史回归。

6. **BVC Level 2 的有效使用率**：V2.1 引入 `|dm| > 2 × TICK_SIZE` 阈值后，Level 2 在做市商活跃品种中可能被频繁过滤，实际退化为 Level 1 + Level 3 双级模式。这是安全性与信号覆盖率的有意权衡。

7. **冷启动与截面排名的交互影响**：V2.1 将冷启动期的特征输出改为 NaN（而非静默默认值），截面 rank 会自动剔除 NaN ETF。但当截面中大量 ETF 处于冷启动期时，有效截面样本量可能不足，导致 rank 分辨率下降。下游应监控 `cold_start` 比例。

8. **两阶段架构约束**：`process_daily` 仅负责 per-ETF 时序标准化，`process_all_etfs` 负责截面 rank。下游消费时序特征不应用于跨 ETF 比较，截面特征（`_xs` 后缀）不应用于时序跟踪。

9. **VWAP 偏离度的无成交区间处理**（V2.1 第三轮）：无成交快照的 VWAP 偏离度输出 NaN，不参与 `vwap_dev_max` 计算。当 `zero_volume_ratio` 过高时，下游应对 VWAP 因子降权或屏蔽。

10. **重复快照去重**（V2.1 第三轮）：M0 新增 `dedup_by_timestamp`，同一时间戳仅保留最后一条。`duplicate_ts_ratio` 超过阈值时全日因子应输出 NaN。

11. **RV Bipower 零成交保护**（V2.1 第四轮）：当有效 `|log_ret|>0` 数量 < 100 时，RV 输出 NaN。零成交段 close 不变导致 log_ret=0 是“无信息”而非“低波动”，下游不应据此低估风险。

12. **尾盘信号稀释与 OFI 子窗口因子**（V2.1 第四轮）：VPIN（50桶窗口）和 Lambda（1h窗口）在尾盘拉升日的信号会被稀释。`ofi_20`/`ofi_60`/`ofi_240` 提供更精细的尾盘信号，下游应结合使用。
