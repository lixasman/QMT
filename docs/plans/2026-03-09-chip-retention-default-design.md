# 筹码数据默认保留三年设计

## 背景

`python -m etf_chip_engine.daily_batch --date auto` 当前默认会按自然日清理历史筹码与相关缓存数据，代码默认值为 `730` 天。当前需求是将默认保留窗口调整为三年，即 `1095` 个自然日。

## 目标

- 将默认数据保留窗口从 `730` 调整为 `1095`
- 保持现有“按自然日清理”的行为不变
- 同步修正文档中的默认值说明
- 用最小改动覆盖默认值相关测试

## 方案对比

### 方案 A（采用）

仅调整默认保留天数常量与文档说明，清理算法保持自然日口径。

- 优点：改动面最小，风险最低，完全符合当前需求
- 缺点：仍不是按交易日计数

### 方案 B

把清理逻辑改为“按交易日保留三年”。

- 优点：更贴近交易数据语义
- 缺点：需要改造截止日期计算与验证逻辑，超出本次范围

## 影响范围

- `etf_chip_engine/daily_batch.py`
- `etf_chip_engine/README.md`
- `tests/test_etf_daily_batch_date.py`
- `tests/test_etf_data_retention.py`

## 验证方式

- 默认值测试应断言 `1095`
- README 默认说明应与代码一致
- 仅运行与默认保留天数相关的现有测试
